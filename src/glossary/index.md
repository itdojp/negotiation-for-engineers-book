# 用語集

本書で頻出する用語を、読者が迷いやすい順に整理する。ここで扱うのは、本書の理解と実務への適用に直結する最小セットである。

## 交渉の基本語彙

| 用語 | 意味（本書での扱い） | 関連 |
|---|---|---|
| 立場（Position） | 表に出ている要求。例: 期限延長、予算増など。 | [基礎語彙](../foundations/) |
| 利害（Interest） | 立場の背後の理由・制約・期待。要求ではなく「なぜ」を扱う。 | [基礎語彙](../foundations/) |
| BATNA | 不合意時に実行できる選択肢のうち、最も望ましい代替案。最低受容ラインや最悪ケースとは区別する。 | [基礎語彙](../foundations/#batna-and-reservation-line) / [付録：実践ツールキット](../appendices/toolkit/) |
| 最低受容ライン | 提案とBATNAを比較した結果として、これを下回るなら合意しないと決める境界。 | [基礎語彙](../foundations/#batna-and-reservation-line) |
| ZOPA | 当事者の最低受容ラインが重なる合意候補の範囲。事前には仮説であり、条件設計で成立を保証できない。 | [基礎語彙](../foundations/#zopa-boundary) |
| アンカリング | 不確実な数値判断で、初期値からの調整が不十分になり得る現象。相手を操作する手法としては扱わない。 | [基礎語彙](../foundations/#anchoring-boundary) |
| 心理的安全性 | 対人的リスクを取っても安全だというチームに共有された信念。快適さ、無条件の同意、低い品質基準とは異なる。 | [基礎語彙](../foundations/#psychological-safety-boundary) |
| 譲歩設計 | 何を・どの順番で・どの条件付きで譲るかを事前に設計すること。 | [基礎語彙](../foundations/) / [第3章](../chapter-3/) |
| 交渉倫理ゲート | 相手の自律性、透明性、権限境界、記録の4点を事前確認するレビュー観点。 | [基礎語彙](../foundations/) / [付録：実践ツールキット](../appendices/toolkit/) |
| 相手の自律性 | 相手が受ける、断る、持ち帰る、別案を出す選択肢を持つ状態。 | [基礎語彙](../foundations/) |

## 本書のメタファ（エンジニアリング由来）

| 用語 | 意味（本書での扱い） | 主な登場箇所 |
|---|---|---|
| ステークホルダー・インターフェース | 相手の関心・判断軸・制約に合わせて、説明粒度と形式を設計すること。 | [第2章](../chapter-2/) |
| 交渉パイプライン | 交渉を「準備→実行→フォロー→改善」の工程として運用する考え方。 | [第6章](../chapter-6/) |
| エモーショナル・デバッグ | 感情やストレスを「観測して調整する対象」として扱う考え方。 | [第5章](../chapter-5/) |
| 反復的合意形成 | 一括で決めず、段階的に合意点を更新し続けるアプローチ。 | [第3章](../chapter-3/) |

## 実務上の成果物（本書が推奨する形）

| 用語 | 意味（本書での扱い） | 関連 |
|---|---|---|
| 交渉準備テンプレート | 交渉前に、背景・利害・代替案・リスク・進め方を構造化する YAML。 | [付録：実践ツールキット](../appendices/toolkit/) |
| 交渉ログ | 交渉の事実（何が起きたか）と次の打ち手を残すMarkdown。継続改善の材料。 | [付録：実践ツールキット](../appendices/toolkit/) |
| 合意のDone | 何が決まれば完了か、誰が決めるか、どう残すかの定義。 | [基礎語彙](../foundations/) |
| 決定記録 | 意思決定者、承認範囲、証跡場所、再確認条件を残す記録。 | [基礎語彙](../foundations/) / [第6章](../chapter-6/) / [付録：実践ツールキット](../appendices/toolkit/) |
