# 付録：交渉トラブルシューティングフロー

このページは、交渉がうまく進まないときに、章を読み直す前の実務入口である。症状を一つ選び、**最初の確認 → 次の行動 → 停止／エスカレーション → 記録**の順に進める。相手を操作するためではなく、事実・制約・権限・責任を安全に揃えるためのフローである。

## 共通フロー

1. **症状を一つに絞る**：いま止まっている現象を、観測できる言葉で書く。
2. **最初の確認をする**：事実、判断者、期限、制約、合意条件を確認する。推測を事実として扱わない。
3. **次の行動を一つ選ぶ**：下表の関連章・ツール・ケースから、最小の検証または会話を実行する。
4. **停止／エスカレーションを判断する**：権限、機密性、安全性、自律性、期限の境界を越えるなら止める。
5. **記録する**：判断、未解決事項、担当者、期限、再確認条件を [交渉ログ](../toolkit/) に残す。

「確認できない」「判断者が不明」「相手が拒否・保留できない状態」「事実と推測を分けられない状態」のいずれかなら、行動を急がず、責任者または適切なレビュー先へ相談する。

## 症状から選ぶ

| 症状 | 最初の確認 | 次の行動 | 停止／エスカレーション | 記録先 |
|---|---|---|---|---|
| 交渉の目的・論点が散らばり、準備を始められない | 目的、判断者、期限、BATNA、最低受容ラインが一文で書けるか | [第6章](../../chapter-6/) の準備フェーズに沿って、[ツールキット](../toolkit/) のテンプレートを最小限埋める | 目的が複数で優先順位を決められない、または自分に決裁権がない場合は責任者へ確認 | 目的・制約・未確定事項を交渉ログへ |
| 技術的に正しい提案が採用されない | 相手の判断軸がコスト、期限、リスク、顧客価値のどれか。提示した根拠は観測事実か | [第1章](../../chapter-1/) で根拠をプロトタイプ／データに変換し、[第4章](../../chapter-4/) で価値・リスク・投資条件を整理する | 判断者、評価基準、必要な承認経路が不明なまま提案を重ねない。評価基準を合意できなければ上位の意思決定者へ | 提案版、根拠、反論、判断基準をログへ |
| ステークホルダーごとに要求や反応が食い違う | 役割、関心、制約、権限、事実と推定の区別を確認する | [第2章](../../chapter-2/) で相手ごとのインターフェースを設計し、[ケース集](../scenarios/) の類似例で問いを作る | 権限外の約束や機密情報の共有を求められたら止め、法務・管理責任者などへ相談 | ステークホルダーマトリクスと未解決の問いをログへ |
| 合意が何度も先送りされる | 未合意の条件、判断期限、次に決める最小単位、最低受容ラインを確認する | [第3章](../../chapter-3/) の段階的合意に切り替え、[ケース集](../scenarios/) のスコープ調整例からPhase／スプリント単位に分ける | 条件を下げても責任・安全性・自律性を損なう場合は譲歩せず、決裁者またはファシリテーターへ | 決定事項、保留理由、次回判断日、担当者をログへ |
| 感情的な対立で事実確認ができない | 会話の安全性、感情の強度、時系列上の事実、参加者の合意を確認する | [第5章](../../chapter-5/) のセルフチェックと再構成を使い、必要なら休憩後に論点を一つだけ再開する | 威圧、差別、報復の懸念、判断能力を失うほどの負荷がある場合は中断し、上長・人事・相談窓口などへ | 事実／解釈／感情／次回の実験を分けてログへ |
| 合意後に実行されない、または同じ問題が再発する | Done、責任者、証跡、監視指標、再確認日、変更時の通知先を確認する | [第6章](../../chapter-6/) のフォローアップと[ツールキット](../toolkit/) のログで、実行単位とレビュー条件を定義する | 責任分界や承認がないまま作業を開始しない。運用責任者または変更管理のレビューへ | 合意ログ、アクション、期限、指標、再発条件を更新 |

## 停止／エスカレーションの共通基準

次のいずれかに該当したら、結論を急がず「何が不足しているか」を記録して相談する。

- **権限**：自分または相手が、その条件を決める権限を持っていない。
- **透明性**：事実、推定、希望条件、未確定条件を分けられない。
- **自律性**：拒否・保留・別案提示の時間や選択肢を残せない。
- **安全性・機密性**：法務、セキュリティ、人事、個人情報、契約などの専門判断が必要である。
- **実行可能性**：責任者、期限、証跡、再確認条件のいずれかがない。
